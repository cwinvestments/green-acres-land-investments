## GREEN ACRES PLATFORM - FULL TECHNICAL OVERVIEW

### What We Built
A comprehensive land financing business management platform for purchasing raw land at auctions and reselling with flexible owner financing.

### Business Model
- Purchase land at auctions (eBay, county auctions)
- Resell with owner financing: $99 down @ 18% APR to 50% down @ 8% APR
- Property range: $2,000-$10,000
- Target customers: Flexible terms without traditional credit requirements
- States: Arizona, Colorado, Arkansas (expanding)
- Tagline: "Your Land. Your Terms"

### Technical Stack

**Frontend:**
- React 18
- Deployed on Netlify (auto-deploy from GitHub)
- Responsive design (desktop 1920x1080, mobile 375px)
- Cloudinary for image management

**Backend:**
- Node.js with Express
- Deployed on Railway (auto-deploy from GitHub)
- RESTful API architecture

**Database:**
- PostgreSQL via Supabase
- Complex relational schema with properties, customers, loans, payments

**Payment Processing:**
- Square API integration
- One-time and recurring payments
- Payment links and invoicing

**Authentication:**
- JWT-based admin authentication
- Secure customer accounts with bcrypt password hashing

## TECHNICAL IMPLEMENTATION NOTES

### Creating Templates from Green Acres

**Steps to Templatize:**

1. **Create Clean Repo**
```bash
   cd C:\Projects
   mkdir OwnerFinancingTemplate
   cd OwnerFinancingTemplate
   git init
```

2. **Copy Green Acres Files**
```bash
   # Copy but exclude:
   # - node_modules
   # - .env files
   # - Database dumps
   # - Specific customer data
```

3. **Environment Variables Template**
   Create `.env.example`:
```
   # Database
   DATABASE_URL=postgresql://user:password@host:5432/database

   # Square
   SQUARE_ACCESS_TOKEN=your_square_token
   SQUARE_ENVIRONMENT=sandbox

   # JWT
   JWT_SECRET=your_secret_key

   # Cloudinary
   CLOUDINARY_CLOUD_NAME=your_cloud_name
   CLOUDINARY_API_KEY=your_api_key
   CLOUDINARY_API_SECRET=your_api_secret

   # App Config
   BUSINESS_NAME=Your Business Name
   ADMIN_EMAIL=admin@yourbusiness.com
```

4. **Configuration File**
   Create `config/business.js`:
```javascript
   module.exports = {
     businessName: process.env.BUSINESS_NAME || 'Your Business',
     loanTerms: {
       downPaymentOptions: [5, 10, 20, 30, 50],
       interestRates: {
         5: 18,
         10: 15,
         20: 12,
         30: 10,
         50: 8
       }
     },
     propertyTypes: ['Land', 'House', 'Commercial'],
     // ... other configurable options
   };
```

5. **Replace Hardcoded Values**
   Find and replace:
   - "Green Acres Land Investments" → `config.businessName`
   - Specific loan terms → `config.loanTerms`
   - Email addresses → `config.adminEmail`
   - Colors/branding → CSS variables

6. **Database Schema**
   Keep generic table/column names:
   - `properties` (not `green_acres_properties`)
   - `customers` (not `green_acres_customers`)
   - Etc.

7. **Railway Template File**
   Create `railway.json`:
```json
   {
     "build": {
       "builder": "NIXPACKS"
     },
     "deploy": {
       "startCommand": "npm start",
       "restartPolicyType": "ON_FAILURE"
     }
   }
```

8. **Docker Support**
   Create `Dockerfile` and `docker-compose.yml` for local development

9. **Documentation**
   - README.md (overview, quick start)
   - INSTALLATION.md (detailed setup)
   - CONFIGURATION.md (all options)
   - API.md (API documentation)
   - TROUBLESHOOTING.md (common issues)

10. **Deploy Instructions**
    Step-by-step for non-technical users:
    - Click Railway template button
    - Connect GitHub account
    - Add environment variables
    - Wait for deployment
    - Configure business settings
    - Add first property/customer

### Testing Strategy

**Before Release:**
1. Fresh database setup
2. Test all user flows
3. Verify all configuration options work
4. Test on Railway (not just local)
5. Have non-technical person test installation
6. Document any issues they encounter
